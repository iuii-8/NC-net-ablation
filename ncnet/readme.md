数据处理说明文档
## 1. 原始数据分析与字段映射
对输入的 `train.csv`、`dev.csv`、`test.csv` 进行了结构对齐，核心提取字段映射如下：
- **模型输入 (Input)**: 映射自原始 CSV 的 `question` 列（用户自然语言查询）。
- **辅助信息 (Schema)**: 映射自 `mentioned_columns` 列（涉及的数据列名）。
- **抽取源 (Source Syntax)**: 映射自高度序列化的 `vega_zero` 可视化语法列。

## 2. 模板标签抽取逻辑
从 `vega_zero` 字符串中通过正则与Token匹配，还原了 6 类多任务核心标签：
1. **mark** (多分类): 提取图表类型（如 `bar`, `line`, `pie`, `scatter`, `none`）。
2. **aggregate** (多分类): 提取聚合函数（如 `count`, `mean`, `sum`, `min`, `max`, `none`）。
3. **filter** (二分类): 识别原句中是否存在过滤条件（`yes`, `no`）。
4. **group** (二分类): 识别是否存在分组横轴（`yes`, `no`）。
5. **sort** (多分类): 识别排序规则（`asc`, `desc`, `none`）。
6. **bin** (二分类): 识别是否包含分箱操作（`yes`, `no`）。

## 4. 规则模板预测器设计
- **匹配原理**: 捕获自然语言中的高频弱特征词（例如：出现 *"rank in descending"* 触发 `sort: desc`；出现 *"grouped by"* 触发 `group: yes`）。
- **生成格式**: 自动将预测的 6 类标签拼装成标准 Token 串：
  `<PT> mark [X] aggregate [X] group [X] filter [X] sort [X] bin [X] </PT>`
- **应用场景**: 脚本已将预测出的模板前缀注入到测试集中，生成 `test_with_rule_template.csv`，可用于进行原始 `ncNet` 的 Baseline 效能测试。
